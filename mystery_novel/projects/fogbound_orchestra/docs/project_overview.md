## 톤 & 장르 믹스
- 미스터리 + 고딕 호러의 질감, 그러나 선택지가 드라마적 공감/선택에 영향
- 배경 사운드(빗방울, 금속 울림, 현악 잔향)를 단서로 서술

## 플레이 구조
- 5막 구조 + 프롤로그/에필로그
- 각 막은 "기억 상태"를 변경하는 장면 포함: clear → hazy → echoed → false → restored
- Branch: `truth_path`(증거 확립) vs `cover_path`(권력 개입) vs `false_memory`(조작 굴복)

## 시스템 아이디어 (XML 맵핑 힌트)
- `<memory_state>` 태그로 현재 기억 신뢰도를 마킹 → 대사/묘사 색상/음향 변조
- `<choice>`에 `requires` 속성으로 능력치 조건, `effects`로 기억 상태 전환
- `<scene>` `next` 속성에 분기 라벨 → LLM이 XML 생성 시 자동 연결 가능
